========================================
  HAWK MOTH — ANIMATION PROMPTS
  For Unity AI Animation Generation
========================================

MODEL INFO
----------
Model:          Meshy AI "Nocturnal Sentinel" (FBX)
Rig:            Generic
Scale:          80 (import), prefab visual scale 2x
Orientation:    FBX exports Z-up; runtime applies -90X correction
Unit Role:      Support caster, healer — no attack animations needed
Movement:       Has both ground and air modes (MothStealth state machine)

CRITICAL REQUIREMENTS
---------------------
1. WING BONES MUST BE NAMED: "L_wing" / "R_wing" or "wing_L" / "wing_R"
   (so the runtime procedural fallback can find them).
2. The moth MUST look like it is FLYING, not jumping or bouncing.
   Wing motion is the primary visual — wings must flap smoothly.
3. Flying height is handled by code (MothStealth component). Animations
   should NOT include vertical root translation.
4. Team color tinting is applied via MaterialPropertyBlock at runtime.
   The model's base material should remain neutral/white for proper tinting.

IMPORTANT NOTES FOR ALL ANIMATIONS
-----------------------------------
- The model faces +Z in its local space (after -90X correction at runtime).
- All animations must loop seamlessly EXCEPT Death, Landing, Takeoff.
- Root motion is NOT used — the NavMeshAgent drives translation.
- Keep all animation on the skeleton; do NOT bake root translation.
- Wing bones are the primary expressive element for this unit.
- The moth is delicate and graceful — all motion should feel light and airy,
  never heavy or mechanical.
- The moth has large, broad wings (like a hawk moth / sphinx moth).
  Wing beats are NOT fast hummingbird flutters — they are broad, sweeping
  strokes that you can actually SEE.

FLYING ANIMATIONS (wings active, airborne)

========================================

--- IDLE_FLY ---
Clip Name:      Idle_Fly
Duration:       1.5-2.0 seconds (looping)
Loop:           Yes

Prompt:
"A large hawk moth hovering in place. Broad wings beat in smooth, sweeping
 up-and-down strokes at roughly 4-6 beats per second. Downstroke is faster
 and more powerful than upstroke. Wings spread wide on downstroke, fold
 slightly on upstroke. Body hangs still and stable beneath the wings with
 only a very subtle vertical bob. Legs tucked close to body. Feathery
 antennae swept back gently. Abdomen hangs down, slight sway.

- Body/thorax: near-still, <5% vertical bob relative to wing motion
- Legs: tucked, occasional micro-twitch
- Antennae: gentle flutter swept back
- Abdomen: slight pendulum sway

Feel: Effortless hovering. Ethereal and graceful. NOT bouncing/jumping.


--- WALK_FLY (Move_Fly) ---

- Legs: pressed tight to body

========================================
ANIMATOR CONTROLLER SETUP
========================================

Controller: Assets/_InsectWars/Animations/HawkMoth.controller

Parameters:
- Speed (Float)       — planar velocity magnitude
- IsMoving (Bool)     — true when unit has movement orders
- IsFlying (Bool)     — true when MothStealth state is Flying/Landing
- Death (Trigger)     — triggers death animation

State Machine:
  Default → Idle_Fly  (moth spawns airborne)

  Idle_Fly → Walk_Fly:        IsMoving == true
  Walk_Fly → Idle_Fly:        IsMoving == false

  Idle_Fly → Landing:         IsFlying == false
  Landing → Idle_Ground:      (has exit time = true, exitTime = 0.9)

  Idle_Ground → Walk_Ground:  IsMoving == true, IsFlying == false
  Walk_Ground → Idle_Ground:  IsMoving == false

  Idle_Ground → Takeoff:      IsFlying == true
  Takeoff → Idle_Fly:         (has exit time = true, exitTime = 0.9)

  Any State → Death:          Death trigger (has exit time = false)

Transition durations: 0.1-0.15s for locomotion blends.
Takeoff/Landing are one-shot states with exit time.

--- PROMPT A: IDLE HOVER (primary — generate this first) ---

Subject:
A large hawk moth / sphinx moth 3D model. It has two broad triangular
wings attached to a fuzzy thorax, six small legs, two feathery antennae,
and a tapered abdomen. The wings are the largest part of the body —
each wing is roughly as long as the entire body.

Animation requested — "Idle Hover" (looping, 1.5–2.0 seconds):
The moth hovers in place, suspended in the air. The WINGS are the
primary moving element — everything else is nearly still.

Wing motion (CRITICAL — this is the whole point of the animation):
- Both wings flap up and down in smooth, broad strokes.
- The flap arc is approximately 50-70 degrees total (from above
  horizontal to below horizontal).
- Flap rate: about 4-6 full cycles per second (fast enough to look
  like flight, slow enough that each beat is visible — NOT a blur).
- The downstroke is slightly faster and more forceful than the upstroke.
- Wings sweep forward slightly on the downstroke, back on the upstroke
  (figure-8 tip path when viewed from the side).
- Wings should NEVER clip through the body or each other.

Body:
- The thorax/body hangs stable beneath the wings. Almost no movement.
- Allow only a very subtle vertical bob (2-3% of body height) synced
  to the wing beats — body dips very slightly on the downstroke.
- NO lateral sway. NO rotation. NO bouncing. The body is a stable
  anchor while the wings do all the work.

Legs: Tucked close to the body, dangling naturally. No movement.
Antennae: Swept back gently. Optional: very subtle flutter.
Abdomen: Hangs down from the thorax. Optional: tiny pendulum sway.

Overall feel:
Effortless, graceful hovering. The moth looks perfectly at home in
the air. Think of a hummingbird hawk-moth hovering at a flower — 
smooth, continuous wing beats with an almost-still body. The animation
should NOT look like jumping, bouncing, or bobbing. The WINGS create
the flight, not the body.

Technical constraints:
- Looping: The last frame must blend seamlessly back to the first.
- No root translation: The moth stays in place (movement is handled
  by game code). Do NOT animate the model moving up/down/sideways.
- No root rotation: The moth faces forward the whole time.
- Export format: FBX with embedded animation clip, or separate FBX
  animation clip referencing this skeleton.

========================================
  TROUBLESHOOTING — WHY WING FLAP FAILS
========================================

Common reasons AI animation tools fail to animate moth wings:

1. SINGLE MESH / NO SKELETON
   The Meshy "Nocturnal Sentinel" model is likely a single fused mesh
   with no bones. Animation tools need separate bones or at least
   separate mesh parts to move the wings independently.
   → Solution: Ask the tool to add a skeleton first, or manually
     split the wings in Blender before sending to the animation tool.

2. TOOL DOESN'T UNDERSTAND "FLAP"
   Many AI animation tools are trained primarily on bipedal/quadruped
   walk cycles. Insect wing flapping is outside their training data.
   → Solution: Use Prompt D (separate-wings approach) or describe
     the motion as "arms waving up and down" if the tool understands
     humanoid motions better.

3. SCALE / ORIENTATION ISSUES
   The FBX exports Z-up. If the tool expects Y-up, the moth may
   appear lying on its back and the "flap" goes sideways.
   → Solution: Specify "Y-up" orientation in the tool settings,
     or rotate the model 90 degrees on X before sending it in.

4. FALLBACK — PROCEDURAL ANIMATION IN UNITY
   If no tool can produce a proper wing-flap clip, the game already
   has a procedural fallback in UnitAnimationDriver.ApplyHawkMothLoop()
   that simulates wing flaps by scaling the model (squash & stretch).
   This looks acceptable from the RTS camera distance but is not
   a true wing animation. To improve it:
   - Open the moth FBX in Blender
   - Separate the two wing meshes (select wing faces → P → Separate)
   - Name the new objects "L_wing" and "R_wing"
   - Add an Armature with two bones at the wing roots
   - Parent the wing meshes to their respective bones
   - Export as FBX
   - UnitAnimationDriver will find the wing bones automatically
     and can drive them procedurally with proper rotation

========================================
  BLENDER MANUAL WORKFLOW (LAST RESORT)
========================================

If all AI tools fail, here's a step-by-step for Blender:

1. Import: File → Import → FBX → select the Meshy model
2. Fix orientation: If Z-up, apply -90X rotation (Ctrl+A → Rotation)
3. Enter Edit Mode on the mesh
4. Select the LEFT wing faces (use L to select linked, or manual box select)
5. Separate: P → Selection. Rename the new object "L_wing"
6. Repeat for the RIGHT wing → name it "R_wing"
7. Remaining body mesh → rename to "Body"
8. Add Armature: Shift+A → Armature
9. In Edit Mode on the armature, create 3 bones:
   - "Root" at the center of the thorax
   - "L_wing" starting at the left wing root, extending into the wing
   - "R_wing" starting at the right wing root, extending into the wing
   - Parent both wing bones to Root
10. Parent each wing mesh to its bone: select mesh, then bone,
    Ctrl+P → Bone (keep offset)
11. Animate:
    - Frame 1: Wings horizontal (rest pose). Insert keyframe (I → Rotation)
    - Frame 4: Wings rotated -35° on local X (downstroke). Keyframe.
    - Frame 10: Wings rotated +35° on local X (upstroke). Keyframe.
    - Frame 12: Back to rest. Keyframe.
    - This gives ~12 frames per beat. At 24fps = 2 beats/sec.
      At 60fps, use 10-15 frames per beat for 4-6 beats/sec.
    - Set interpolation to sinusoidal (Graph Editor → T → Sinusoidal)
12. Set frame range, mark as looping (Shift+E → Make Cyclic)
13. Export: File → Export → FBX
    - Check "Armature" and "Mesh" in export filter
    - Animation: check "Baked Animation" and "NLA Strips"
    - Apply Transform: checked
    - Forward: -Z Forward, Up: Y Up (Unity convention)
